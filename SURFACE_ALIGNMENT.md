# Surface Alignment Guide

Use this guide to keep GitHub, website, and Discord fully aligned.

Canonical references:

- [Foundation entry point](./README.md)
- [Canonical sources](./CANONICAL_SOURCES.md)

## Canonical Message

Use this sentence verbatim across all surfaces:

> The UOR Foundation is the steward of the Universal Object Reference (UOR) framework. We support open-source projects built on top of UOR through governance, lifecycle review, standards, and contributor support.

## Canonical Lifecycle Labels

These are the only lifecycle stages:

1. Sandbox
2. Incubation
3. Graduated
4. Archived

Reference: [TOC process](./toc/README.md).

`uor-adjacent` may be used as a discussion tag in Discord, but it is not a lifecycle stage.

## Canonical Roles

Use governance roles from the framework:

- Organisation Owner
- Repository Maintainer
- Core Contributor
- External Contributor
- Automated Agent

Reference: [Governance roles](./governance/GOVERNANCE.md#41-roles)

If Discord uses community-facing labels (for example, "community member"), map them to governance roles in `#start-here`.

## Discord Notes

- `#start-here` should include: canonical message, server map, lifecycle labels, and canonical policy links.
- `#library` should store links to canonical GitHub documents, not duplicate policy text.
- `#library` should include a dedicated "Framework Source" link to [UOR-Foundation/UOR-Framework](https://github.com/UOR-Foundation/UOR-Framework).
- `#governance` should publish decisions and link back to canonical policy documents.
- `#funding` should avoid statements that conflict with repository licensing policy. Current canonical licensing reference: [GOVERNANCE.md Section 13](./governance/GOVERNANCE.md#13-intellectual-property--licensing).

## Rule

If documents conflict, the canonical source listed in `CANONICAL_SOURCES.md` prevails.
